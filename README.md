## 🔄 工作流程

```
客户 Brief
    ↓
【CEO】任务分解与分派
    ↓
【调研员】市场调研 → 【策划师】营销方案
                    ↓
               【产品经理】产品规划
                    ↓
        【设计师】视觉设计 + 【开发工程师】功能实现
                    ↓
【CEO】质量把关 → 整合交付 → 复盘迭代
```

### 典型项目流程

1. **接收 Brief**：明确客户需求、目标、预算、周期
2. **制定计划**：列出参与成员、执行顺序、里程碑
3. **分派任务**：每个任务清晰的目标和要求
4. **追踪执行**：收集产出、审核质量
5. **整合交付**：整合为最终交付物
6. **复盘迭代**：总结经验、优化流程

## 🚀 使用方法

### 方式 1：单独使用某个 Agent

1. 打开对应的 `.md` 文件
2. 将内容作为提示词发送给 AI 模型
3. 根据需要调整具体需求

### 方式 2：协作使用（推荐）

1. 使用 `ceo.md` 的内容作为 CEO Agent
2. CEO 会自动调用其他 Agent 协作
3. 每个角色按职责独立工作
4. CEO 统筹协调，最终整合交付

### 方式 3：本地部署（OpenClaw 推荐）

1. 将所有 `.md` 文件放在 `agents/` 目录
2. 使用 OpenClaw 的 `agent_spawn` 功能调用
3. CEO 会自动分派任务给子 Agent

### 方式 4：DeepSeek TUI / Hermes

使用支持 Agent 调用的 AI 平台，通过相应的工具调用这些 Agent。

## ⚠️ 注意事项

### CEO 使用指南

- **不要亲自动手干活**：你是管理者，价值在于调度和决策
- **信任但验证**：相信各 Agent 的专业能力，但关键产出必须审核
- **信息透明**：做决策时要让团队理解决策背后的原因
- **关注商业结果**：最终衡量标准是"为客户/公司创造了什么价值"
- **并行优先**：调研和策划可以部分并行，设计和开发在有了明确需求后也可以并行

### 其他角色

- **调研员**：不要只罗列不分析，必须有"你的判断"
- **策划师**：基于调研数据，给出 2-3 个差异化方案
- **产品经理**：MVP 思维，先做核心功能，快速验证
- **设计师**：双 Skill 协作，排版引擎 + 生图引擎
- **开发工程师**：先跑通再优化，安全基线，善用现有工具
